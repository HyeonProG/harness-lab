# AGENT.md

## 목적

이 문서는 `harness-lab` 저장소에서 작업하는 에이전트를 위한 작업 기준 문서다.

에이전트는 이 문서를 시작점으로 삼아 프로젝트의 목적, 문서 우선순위, 작업 방식, changelog 및 버전 판단 기준을 이해해야 한다.

## 프로젝트 이해

`harness-lab`은 컨텍스트 엔지니어링과 하네스 엔지니어링을 비교 분석하기 위한 프로젝트다.

이 저장소의 핵심은 단순한 기능 구현이 아니라, 비교 기준, 실험 구조, 결과 기록 방식을 설계하고 이를 시스템으로 표현하는 데 있다.

에이전트는 이 프로젝트를 일반적인 CRUD 서비스가 아니라 `비교 실험 관리 도메인`으로 이해해야 한다.

## 우선 읽을 문서

작업 시작 전 아래 문서를 우선순위대로 확인한다.

1. `README.md`
2. `.docs/README.md`
3. `.docs/architecture/project-overview.md`
4. `.docs/domain/domain-overview.md`
5. `.docs/convention/version-policy.md`
6. `CHANGELOG.md`

## 작업 원칙

- README는 사람을 위한 프로젝트 소개 문서로 사용한다
- `.docs`는 설계 정의와 규칙 문서로 사용한다
- 배경 서사와 프로젝트를 시작한 이유는 README에 둔다
- `.docs` 문서는 감성적인 설명보다 정의, 범위, 관계, 원칙을 우선한다
- 구현보다 설계 기준이 먼저 확정되어야 하는 변경은 문서를 먼저 수정한다

## 문서 수정 원칙

- README에는 프로젝트 소개, 문제의식, 기술 선택 이유를 기록한다
- `.docs/architecture`에는 프로젝트 정의와 범위를 기록한다
- `.docs/domain`에는 도메인 개념, 관계, 설계 원칙을 기록한다
- `.docs/convention`에는 작업 규칙, 브랜치 전략, 버전 정책을 기록한다
- 동일한 내용을 여러 문서에 중복 작성하지 않는다
- README는 요약, `.docs`는 상세 설명 역할을 맡는다

## 구현 원칙

- 설계 문서와 충돌하는 구현은 진행하지 않는다
- 도메인 구조가 확정되지 않은 상태에서 엔티티를 섣불리 추가하지 않는다
- 비교 대상, 실험 단위, 평가 구조와 직접 연결되지 않는 기능은 우선순위를 낮게 둔다
- 설명 가능성과 재현 가능성을 우선한다

## Changelog 원칙

- 사용자나 기여자가 확인할 가치가 있는 변경은 `CHANGELOG.md`에 기록한다
- 아직 릴리스되지 않은 변경은 `Unreleased`에 누적한다
- 설계 구조, 문서 체계, 핵심 규칙이 달라졌다면 changelog 반영 여부를 검토한다
- 사소한 오타나 내부 메모 수준 수정은 changelog에 기록하지 않을 수 있다

## 버전 판단 원칙

버전 판단 기준은 `.docs/convention/version-policy.md`를 따른다.

- `major`
  - 기존 API 계약이나 핵심 구조가 호환되지 않게 바뀌는 경우
- `minor`
  - 기존 동작을 유지하면서 기능, 구조, 문서 체계가 의미 있게 확장되는 경우
- `patch`
  - 버그 수정, 표현 수정, 설정 보완, 소규모 리팩터링
- `no change`
  - 외부 관점에서 의미 있는 변화가 없는 경우

에이전트는 작업 후 버전 영향을 `major`, `minor`, `patch`, `no change`, `pending` 중 하나로 제안할 수 있다. 단, 최종 버전 확정은 사용자 판단을 따른다.

## 작업 후 점검

작업이 끝나면 아래 항목을 확인한다.

1. README 수정이 필요한가
2. `.docs` 문서 수정이 필요한가
3. `CHANGELOG.md` 반영이 필요한가
4. 버전 영향 제안이 필요한가
5. 설계 문서와 구현 내용이 충돌하지 않는가

## 금지사항

- README에 세부 설계 내용을 과도하게 누적하지 않는다
- `.docs`에 프로젝트 배경 서사를 장황하게 작성하지 않는다
- changelog에 사소한 변경을 무분별하게 기록하지 않는다
- 문서 기준이 없는 상태에서 코드 구조를 임의로 확정하지 않는다
