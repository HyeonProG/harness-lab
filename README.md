# Harness Lab

## 프로젝트 소개

`harness-lab`은 컨텍스트 엔지니어링과 하네스 엔지니어링을 비교 분석하기 위한 개인 프로젝트

이 프로젝트는 두 개념을 설명하는 데서 끝나지 않고, 동일한 시나리오에서 두 접근 방식을 비교할 수 있도록 실험 구조, 평가 기준, 결과 기록 방식을 설계하고 이를 웹 기반 서비스로 구현하는 것을 목표로 한다.

## 프로젝트를 시작한 이유

실무에서 에이전트를 활용해 개발 작업을 진행하면서, 동일한 에이전트와 유사한 요청을 사용하더라도 항상 동일한 수준의 결과가 나오지 않는 문제를 반복해서 경험했다.

어떤 경우에는 충분한 문맥을 제공했을 때 좋은 결과가 나왔지만, 다른 경우에는 같은 방향의 요청이어도 결과 품질이 흔들리거나 재현성이 떨어졌다. 이 과정에서 단순히 프롬프트를 잘 작성하는 것만으로는 한계가 있고, 작업 흐름에 일정한 제약과 검증, 재시도 구조를 넣는 방식이 필요하다고 판단하게 됐다.

이 프로젝트는 그 경험에서 출발한다. 단순히 컨텍스트를 더 많이 제공하는 방식과, 프레임워크처럼 일정한 제약과 재시도 구조를 부여하는 방식이 실제로 어떤 차이를 만드는지 비교해보고자 한다.

## 프로젝트 목표

- 컨텍스트 엔지니어링과 하네스 엔지니어링의 차이를 명확한 기준으로 정리한다
- 두 접근 방식을 동일한 시나리오에서 비교 가능한 구조로 모델링한다
- 비교 결과를 저장하고 조회할 수 있는 백엔드 구조를 설계한다
- 설계 과정과 변경 이력을 문서와 커밋으로 관리한다

## 프로젝트 성격

이 프로젝트는 일반적인 CRUD 중심 서비스가 아니라 다음 성격을 가진다.

- 실험 관리 도메인
- 평가 및 비교 분석 도메인
- AI 엔지니어링 방법론 비교 도메인

즉, 핵심은 사용자나 상품을 관리하는 것이 아니라 `비교 대상`, `시나리오`, `실험`, `평가`, `근거 데이터`를 구조화하는 것이다.

## 기술 스택

### Backend

- Java 17
  - LTS 버전으로서 안정성이 높고, 최신 문법과 실무 범용성 사이의 균형이 좋다.
  - 개인 프로젝트에서 지나치게 실험적인 버전보다 설명 가능성과 재현 가능성을 우선하기에 적합하다.
- Spring Boot 4.0.5
  - 애플리케이션 설정, 웹 계층, 의존성 구성을 빠르게 표준화할 수 있어 초기 설계 실험에 유리하다.
  - 비교 대상, 시나리오, 실험 결과 같은 도메인을 REST API 중심으로 구조화하기 적합하다.
- Spring MVC
  - 이 프로젝트는 화면보다 API와 도메인 구조가 먼저이기 때문에, 요청과 응답 흐름을 명확하게 구성하는 데 적합하다.
  - 비교 실험 결과를 조회하고 등록하는 서버 구조를 단순하고 설명 가능하게 유지할 수 있다.

### Data

- Spring Data JPA
  - 실험, 평가, 근거 데이터처럼 명확한 관계를 가진 도메인을 빠르게 모델링하기 좋다.
  - 초기 구현 단계에서 반복적인 저장소 코드를 줄이고 핵심 도메인 설계에 집중할 수 있다.
- QueryDSL 6.10.1
  - 비교 결과, 평가 기준, 시나리오 조합처럼 조회 조건이 점점 복잡해질 가능성이 높기 때문에 타입 안전한 동적 쿼리가 필요하다.
  - 문자열 기반 쿼리보다 리팩터링에 강하고, 포트폴리오에서 조회 설계 의도를 설명하기 좋다.
- PostgreSQL
  - 관계형 데이터와 명확한 스키마 제약을 기반으로 도메인 구조를 검증하기에 적합하다.
  - 실험, 평가, 메트릭, 근거처럼 구조화된 데이터를 일관되게 저장하고 조회하기 좋다.

### Build

- Gradle
  - Spring Boot 프로젝트에서 표준적으로 사용하기 좋고, 의존성 및 빌드 설정을 선언적으로 관리할 수 있다.
  - 이후 프론트엔드 통합이나 단일 배포 구조로 확장하더라도 빌드 파이프라인을 유연하게 가져가기 좋다.

## 버전관리 원칙

- 문서와 코드를 함께 관리한다
- 작은 단위의 커밋을 우선한다
- 설계 변경 이유가 드러나는 메시지를 사용한다
- 구현 전에도 문서 변경 내역을 남긴다

예시:

- `docs: README 초안 작성`
- `docs: 프로젝트 개요 문서 추가`
- `docs: 도메인 모델 초안 작성`
- `feat: scenario 엔티티 추가`

## 정리

`harness-lab`은 AI 개발 방법론을 소개하는 프로젝트가 아니라, 두 접근 방식을 비교 가능한 실험 구조와 시스템으로 표현하는 프로젝트를 목표로 한다.

이 저장소는 구현 결과만 담는 공간이 아니라 설계와 판단 기준이 함께 축적되는 포트폴리오 저장소로 관리한다.
