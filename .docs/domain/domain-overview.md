# 도메인 개요

## 도메인 성격

`harness-lab`의 핵심 도메인은 이커머스나 커뮤니티가 아니라 `비교 실험 관리`에 가깝다.

이 프로젝트는 특정 접근 방식을 동일한 조건에서 비교하고, 결과를 해석하고, 근거를 누적하는 구조를 중심으로 설계한다.

## 도메인 분류

- 1차 도메인: 실험 관리
- 2차 도메인: 평가 및 비교 분석
- 주제 특화 영역: AI 엔지니어링 방법론 비교

즉, 이 프로젝트의 핵심은 기능 제공형 서비스가 아니라 비교 구조와 결과 해석 구조를 관리하는 것이다.

## 핵심 개념

### Approach

비교 대상이 되는 접근 방식이다.

### Scenario

비교를 수행하는 과제 단위다.

### Experiment

특정 시나리오를 특정 접근 방식으로 수행한 실행 단위다.

### Evaluation

실험 결과를 기준별로 평가한 해석 정보다.

### Metric

성공 여부, 수정 횟수, 검증 통과 여부처럼 구조화된 결과 값이다.

### Evidence

평가와 결과의 근거가 되는 메모, 로그, 산출물 정보다.

## 도메인 관계

- 하나의 `Scenario`는 여러 `Experiment`를 가진다
- 하나의 `Approach`는 여러 `Experiment`에 연결될 수 있다
- 하나의 `Experiment`는 여러 `Evaluation`, `Metric`, `Evidence`를 가진다

## 도메인 목적

이 도메인 구조는 다음 질문에 답하기 위해 설계한다.

- 같은 시나리오에서 어떤 접근 방식이 더 유리했는가
- 결과 차이는 어떤 기준에서 발생했는가
- 차이를 뒷받침하는 근거는 무엇인가

## 설계 원칙

- 결과와 해석을 분리한다
- 점수만 저장하지 않고 근거를 함께 남긴다
- 초기 버전에서는 비교 대상과 기준을 제한해 설명 가능성을 유지한다
- 자동 실행보다 비교 구조와 결과 관리에 먼저 집중한다
